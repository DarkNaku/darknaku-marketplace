## 3. Dynamics — 게임 루프 & 시스템 흐름

> Mechanics가 실행될 때 나타나는 행동 패턴. 구현보다 "결과"에 집중.

### 3.1 코어 루프 (Core Loop)
```
[세션 내 1회 반복 단위 — 최대한 구체적으로]

예)
① 스테이지 선택 (월드맵 화면)
② 게임플레이 시작 → 퍼즐 조작
③ 클리어: 별 1~3개 획득 + 코인 보상
   실패: 하트 1개 소모 / 광고 시청으로 1회 재시도 가능
④ 결과 화면: 보상 연출 → 다음 스테이지 잠금 해제
⑤ 메인 화면 복귀 → ①로 반복
```

### 3.2 메타 루프 (Meta Loop)
```
[세션 간 장기 진행 — 플레이어를 계속 돌아오게 만드는 구조]

예)
① 코어 루프 반복 → 재화/아이템 누적
② 누적 자원으로 메타 시스템 성장 (강화/수집/언락)
③ 성장 결과가 코어 루프를 더 쉽게 / 새로운 콘텐츠 개방
④ 신규 콘텐츠/이벤트 → 재방문 동기 → ①로 반복
```

### 3.3 세션 구조
| 항목 | 내용 |
|------|------|
| 평균 세션 시간 | (추정) |
| 세션당 플레이 단위 | (예: 스테이지 3~5개) |
| 자연 세션 종료 트리거 | (예: 하트 소진, 목표 달성) |
| 강제 이탈 트리거 | (예: 광고 과다, 에너지 소진) |
| 재방문 유인 | (예: 하트 회복, 일일 보상) |

### 11.1 Mechanic → Dynamic → Aesthetic 매핑

| Mechanic | → Dynamic | → Aesthetic |
|----------|-----------|-------------|
| | | |
| | | |
| | | |

### 11.2 이 게임의 핵심 MDA 사슬
```
[이 게임을 재미있게 만드는 가장 중요한 인과 구조 — 서술형]

예) "{핵심 메커니즘}(Mechanic)이 {행동 패턴}(Dynamic)을 만들고,
    이것이 {감정 경험}(Aesthetic: {유형})을 유발한다.
    클론 개발 시 이 사슬을 유지하는 것이 최우선 목표다."
```

### 11.3 설계 취약점 (의도치 않은 부정적 Dynamic)
```
[Mechanic이 의도한 Aesthetic을 오히려 방해하는 지점]

예) "{광고 강제 시청}(Mechanic)이 {몰입 단절}(Negative Dynamic)을 유발하여
    핵심 Aesthetic인 {Submission/Flow}을 저해할 수 있다.
    클론 개발 시 광고 삽입 타이밍을 세션 경계로 제한하는 것을 권장."
```
