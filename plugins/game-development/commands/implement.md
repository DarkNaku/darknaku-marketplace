# /implement 커맨드

`features.md`의 다음 개발 항목을 가져와 세부 계획을 수립하고, 계획에 따라 개발·테스트까지 완료한다.
추가 내용이 있는 경우 `features.md`를 먼저 업데이트한 뒤 동일하게 진행한다.

---

## 사용법

```
/implement              # features.md의 다음 항목 구현
/implement {기능 설명}  # 해당 내용을 features.md에 추가/수정 후 구현
```

**예시**:
```
/implement
/implement 하트 자동 회복 기능 추가
/implement FEAT-003 설명 수정 후 구현
```

---

## 실행 절차

### Step 1: 입력 분기

`$ARGUMENTS` 값에 따라 분기한다.

**A. 인자 없음**
- `features.md`에서 미완료 항목 중 첫 번째(다음 개발 항목)를 읽는다.
- 미완료 항목이 없으면: "features.md에 미완료 항목이 없습니다. 개발할 기능을 입력해 주세요." 안내 후 종료.
- 항목이 있으면 바로 Step 2로 진행.

**B. 인자 있음**
- `feature` 스킬의 워크플로우를 실행한다.
  - 스킬 파일: `.claude/skills/feature/SKILL.md` — **반드시 먼저 읽고** 해당 스킬 절차를 따른다.
  - 새로운 기능 요청이면 미완료 목록 맨 앞에 추가한다.
  - 기존 항목 수정 요청이면 해당 항목을 수정한다.
- `features.md` 업데이트 완료 후 Step 2로 진행.

---

### Step 2: 구현 대상 확정

Step 1에서 결정된 기능을 사용자에게 알린다:

```
▶ 구현 대상: [{FEAT-NNN}] {기능명}
   {기능 설명}

세부 계획을 수립합니다...
```

---

### Step 3: 세부 계획 수립 (feature-plan 스킬 실행)

`feature-plan` 스킬의 워크플로우 전체를 실행한다.

- 스킬 파일: `.claude/skills/feature-plan/SKILL.md` — **반드시 먼저 읽고** 해당 스킬의 워크플로우(Step 1~5)를 그대로 따른다.
- 인풋: Step 2에서 확정된 기능명 및 설명
- 출력: `features/{FeatureName}.md` 작업 명세서

feature-plan 스킬은 계획 수립 후 사용자 승인을 받는다. 승인 없이 Step 4로 넘어가지 않는다.

---

### Step 4: 계획에 따른 개발 실행

Step 3에서 생성된 작업 명세서의 체크리스트를 **순서대로** 실행한다.

**실행 원칙**:
- 코드 작업은 반드시 **Red → Green → Refactor** 순서를 지킨다.
  - [Red] 테스트를 먼저 작성하고 실패를 확인한다.
  - [Green] 테스트를 통과하는 최소한의 코드를 구현한다.
  - [Refactor] 테스트가 통과된 상태에서만 리팩터링한다.
- 각 태스크 완료 시 작업 명세서의 해당 체크박스를 `[x]`로 업데이트한다.
- 수동 작업 항목이 있으면 사용자에게 안내하고 완료 확인을 요청한 뒤 다음 단계로 진행한다.

**수동 작업 처리**:
```
⚠️ 수동 작업 필요:
   {수동 작업 내용}

완료되면 "완료" 또는 "continue"를 입력해 주세요.
```

---

### Step 5: 테스트 실행 및 확인

모든 태스크 완료 후 테스트를 실행한다.

- **EditMode 테스트**: 먼저 실행 (PlayMode 테스트 제외)
- 테스트 실패 시: 실패 원인을 분석하고 수정한 뒤 다시 실행한다. 반복 실패 시 사용자에게 상황을 보고한다.
- 모든 테스트 통과 확인 후 Step 6으로 진행.

```
✅ 테스트 통과
```

---

### Step 6: features.md 완료 표시

구현된 기능을 `features.md`에 완료 표시한다.

- `feature` 스킬의 "완료 표시" 절차를 따른다.
- 해당 항목: `[ ]` → `[x]`, 목록 하단으로 이동, 날짜 갱신.

---

### Step 7: 완료 요약 출력

```
✅ /implement 완료

🎯 구현 기능: [{FEAT-NNN}] {기능명}
📋 작업 명세서: features/{FeatureName}.md
```

---

## 주의사항

- 수동 작업이 필요한 경우 건너뛰지 않고 반드시 사용자에게 안내한다.
- 각 스킬의 품질 기준을 충족한 후 다음 단계로 넘어간다.
